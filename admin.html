How content updates work

Changes here update the site_content table in Supabase. The homepage reads this on load — visitors see updates within seconds.

If Supabase is unavailable, the site shows its built-in defaults automatically.
Required Supabase tables
site_content (key, value)
site_slides (id, url, type, title, caption, tag_text, tag_type, cta_text, cta_url, overlay_color, sort_order, active, storage_path)
site_testimonials (id, quote, name, role, initial, sort_order, active)
site_updates (id, icon, color, badge, badge_style, ico_style, title, desc, cta, cta_url, num, ver, sort_order, active)
site_ticker (id, text, type, sort_order, active)
Storage bucket: site-media (public)
